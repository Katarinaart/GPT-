# Как закоммитить и смёрджить PR при конфликте

Если в Pull Request нет кнопки **Merge**, значит есть конфликты и GitHub не может
разрешить их в веб-интерфейсе. Нужно сделать это локально, затем запушить
исправления в ту же ветку PR.

## Пошагово

1. Обновите информацию о ветках:
   ```bash
   git fetch origin
   ```
2. Переключитесь на ветку PR:
   ```bash
   git checkout <branch-name>
   ```
3. Влейте основную ветку, чтобы увидеть конфликты:
   ```bash
   git merge origin/main
   ```
4. Откройте конфликтные файлы (например, `index_FINAL_corrected 3.html`) и
   вручную удалите маркеры конфликтов (`<<<<<<<`, `=======`, `>>>>>>>`).
5. Добавьте исправленные файлы в индекс и создайте коммит:
   ```bash
   git add index_FINAL_corrected\ 3.html
   git commit -m "Resolve merge conflicts"
   ```
6. Отправьте обновления в ветку PR:
   ```bash
   git push origin <branch-name>
   ```

После этого конфликтов не будет, и на странице PR появится кнопка **Merge**.

## Если нужно только обновить PR без merge

Чтобы просто «закоммитить» изменения в ветку PR (без мерджа в main), используйте
команды `git add` → `git commit` → `git push` в своей ветке. PR автоматически
обновится.
